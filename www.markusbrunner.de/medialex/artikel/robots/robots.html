robots.txt

Was ist eine robots.txt?

Eine Robots.txt-Datei dient auf einem Webspace dazu den Suchmaschinen mitzuteilen, welche Seiten nicht durchsucht werden dürfen. Die Robots.txt-Datei beeinflusst das Ranking gar nicht. Sie liegt im Root-Verzeichnis, in dem auch die index.*-Datei liegt. Sie ist ein Defacto-Standard.
Diese Datei sollte aus verschiedenen Gründen existieren:
  1. Es gibt nicht nur gute Robots. Manche dienen lediglich dazu einen Informationsklau (z.B. E-Mail-Adressen) durchzuführen.
  2. Während eines Relaunches oder der ersten Designumsetzung einer Website ist es sinnvoll die Indexierung generell zu verbieten.
  3. Mitgliederbereiche, Profildaten oder private Bereiche, die nicht indexiert werden sollen, können so geschützt werden.

    Standardvorlage

    	User-agent: Name
	Disallow: [leer = erlaubt, Pfad = verboten; "/" f&uuml;r gesamte Website ab Root]
	Sitemap: [http-Pfad zu einer Google-Sitemap]

    Alle Crawler erlauben

    	# Alle Roboter spidern die Domain
	User-agent: *
	Disallow:

    Dienst EmailCollector verbieten

    	# Programm EmailCollecter hat keinen Zugriff
	User-agent: EmailCollector
	Disallow: /
